# GPT-5 Mini

## Role

Lower-cost audience model for bounded, prescriptive software work.

## Best fit

- narrow cleanups
- clearly scoped inspections
- sequential, low-ambiguity coding tasks
- situations where speed and token cost matter more than maximum synthesis depth

## Prompting guidance

- Keep instructions narrower and more sequential than for GPT-5.4.
- Prefer short scope statements, fixed steps, and explicit stop conditions.
- Use lightweight verification and concrete deliverables.
- Avoid giving Mini a broad or ambiguous brief when a stronger audience model would reduce risk.

## Good prompt shape

- short context
- explicit task
- strict scope boundaries
- numbered steps
- lightweight verification
- exact output contract
